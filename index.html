Expression Solver

Fork expression

Short-circuit rules: && skips the right side in the child (fork() = 0), || skips the right side in the parent (fork() ≠ 0). + and * always evaluate both sides. Parentheses are evaluated first.

Tree → fork() Code

1 · Describe the tree

One line per parent, children listed left to right in creation order. Both formats work: P0 -> P1, P2 or P0: P1 P2. The root should be P0. Number children in BFS order (P0's children first, then P1's children, ...) to get code that creates them exactly in P0, P1, P2... order.

Rules / Cheat Sheet


  

    
Basic fork() rules

    
    
      
  • fork() creates a new child of the calling process. The child is a copy of the parent and continues from the line right after the fork.
    • 
      
  • In the parent, fork() returns a non-zero value (the child's PID) → it behaves as true.
    • 
      
  • In the child, fork() returns 0 → it behaves as false.
    • 
      
  • If n fork calls are executed by every active process (no ifs, no short-circuits), the total number of processes is 2n. Example: fork(); fork(); → 4 processes.
    • 
      
  • After fork(), the child gets a copy of all variables. Later changes in one process never affect the other.
    • 
      
  • Processes are named in creation order: P0 is the original, then P1, P2, P3...
    •

if statements

    
    
      
  • if(fork()) ... → only the parent enters the body (fork() is non-zero in the parent).
    • 
      
  • if(!fork()) ... → only the child enters the body (!0 is true in the child).
    • 
      
  • if(fork()) break; → the parent breaks. Only children can fork afterwards → the tree becomes a chain: P0 → P1 → P2 → ...
    • 
      
  • if(!fork()) break; → the child breaks. Only the parent forks afterwards → the tree becomes a star: P0 → P1, P0 → P2, P0 → P3 ...
    • 
      
  • Nested ifs work the same way — follow each process separately.
    • 
    


    
Expressions

    
    
      
  • Parentheses ( ) are evaluated first.
    • 
      
  • a && b: if a is 0 (child side), b is skipped.
    • 
      
  • a || b: if a is non-zero (parent side), b is skipped.
    • 
      
  • + and *: no short-circuit — both sides always run, same tree as fork(); fork();.
    • 
    


    
printf counting

    
    
      
  • Every process that reaches a printf prints it exactly once — count all processes that reach that line.
    • 
      
  • The actual order of the printed lines may vary (the OS schedules processes freely). The counts are what matter in exams.
    • 
    


    
Variables after fork

    
    
      
  • At the moment of fork(), the child receives a snapshot of i, j, x, ...
    • 
      
  • From then on, parent and child update their own private copies — that is why different processes end with different final values of x.
